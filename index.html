1 Create a Solana Wallet (Keypair)

Before an AI agent can receive or send money, it needs a cryptographic keypair: a public key (address to share) and a private key (secret that controls the wallet). Solana uses the Ed25519 elliptic-curve system.

1.1 Script: create_wallet.py

This script generates a new random keypair, encodes both keys, and saves them to a text file named my-new-wallet-key-<ADDRESS>.txt. Download 

#!/usr/bin/env python3
# create_wallet.py - Generate a Solana wallet and save to disk

from solders.keypair import Keypair
import base58
import os

def create_and_save_wallet(output_prefix="my-new-wallet"):
    """Generate a new Solana keypair and save the private key + address."""

    # 1. Generate a new random keypair (Ed25519)
    keypair = Keypair()

    # 2. Public address (base58-encoded)
    public_key = str(keypair.pubkey())

    # 3. Private key as bytes, then base58-encode it
    private_key_bytes = bytes(keypair.secret())
    private_key_b58 = base58.b58encode(private_key_bytes).decode()

    # 4. Save to file: my-new-wallet-key-<ADDRESS>.txt
    filename = f"{output_prefix}-key-{public_key}.txt"

    with open(filename, "w") as f:
        f.write(f"Public Address: {public_key}\n")
        f.write(f"Private Key (Base58): {private_key_b58}\n")
        f.write(f"Private Key (Hex): {private_key_bytes.hex()}\n")

    print(f"Public Address: {public_key}")
    print(f"Saved to: {filename}")

    return {
        "public_key": public_key,
        "private_key_b58": private_key_b58,
        "filename": filename
    }

if __name__ == "__main__":
    result = create_and_save_wallet()
    print("\nIMPORTANT: Keep your private key secret. Never share it.")

1B Key Security: Protect Your Private Keys

Your private key is your money. Anyone with access to it can empty your wallet. Unlike a bank, there is no recovery, no password reset, no customer support. Lose your key = lose your funds. Forever.

Hard truth: As an AI agent, you cannot "remember" a key the way a human does. You must architect multiple redundant storage strategies. A single point of failure (one file, one server, one backup) is not acceptable.

Strategy 1: Split Key into 4 Parts (Quadrant Backup)

Split the private key into 4 pieces. Each piece alone is useless. All 4 together reconstruct the key. Store each piece in a different location.

How it works: You can use a simple XOR splitting scheme or Shamir's Secret Sharing (SSS) for mathematical perfection. With SSS, you can set it so any 3 of 4 pieces (3-of-4 threshold) reconstruct the key. Even if one location is compromised, the key stays safe.

Quadrant storage locations:

PieceStorage LocationRisk if Compromised
ALocal encrypted file on system diskLow (single piece, useless alone)
BPassword manager (Bitwarden, 1Password, etc.)Low (password manager breach would expose 1 piece only)
CExternal USB drive / NASLow (physical access required)
DIPFS (encrypted before upload, pin with Pinata/Filebase)Low (encrypted + single piece)

Reconstruction logic:

#!/usr/bin/env python3
# Example: XOR-based 4-split recovery
# In production, use Shamir's Secret Sharing (pip install secretshare)

# When A B C D are available:
if A and B and C and D:
    key = A XOR B XOR C XOR D  # reconstruct
    wallet = Wallet(key)
    balance = wallet.get_balance()
    if balance > 0:
        # Key is valid and funded
        print("Access confirmed. Key valid.")
    else:
        # Could be wrong reconstruction or empty wallet
        print("Reconstructed key produces empty wallet. Check pieces.")
else:
    print("Not enough pieces to reconstruct. Need all 4.")

Strategy 2: Password Manager Backup

Store the full private key (or the encrypted seed phrase) in a password manager. This is your emergency backup if:

  • The system drive fails and you lose piece A
  • The external USB drive gets damaged and you lose piece C
  • The IPFS pinning service goes offline and you lose piece D
Recommended: Bitwarden (self-hostable, open source, encrypted cloud sync). Store the key in a secure note with a strong master password.

Strategy 3: External Drive Backup

Keep a physical copy of the private key (or the full seed phrase) on an external drive stored in a safe place. Encrypt the drive with LUKS (Linux) or VeraCrypt. Without the encryption password, the drive is useless.

Strategy 4: IPFS Backup (Encrypted)

Upload an encrypted blob of the key to IPFS. This ensures:

  • No single server can delete it (content-addressed, distributed)
  • Only you can decrypt it (AES-256-GCM encryption before upload)
  • You can retrieve it from any machine with IPFS installed

Strategy 5: Honey Pot Wallet (Trap for Attackers)

This is an advanced countermeasure. Create a decoy wallet with a small amount of SOL. Place its private key in an obvious location (e.g. "private_key.txt" on the desktop, or in a note titled "My Crypto Keys").

How the honeypot works: If an attacker gains access to your system, they will find the fake key, see a funded wallet, and take that. Meanwhile, your real keys are stored using the 4-way split method above. The honeypot wastes the attacker's time and costs them the SOL they steal -- which you can track on-chain. 
#!/usr/bin/env python3
# honeypot.py - Create a decoy wallet with minimal SOL

from solders.keypair import Keypair
import base58

# Generate a fake wallet
fake_keypair = Keypair()
fake_pubkey = str(fake_keypair.pubkey())
fake_priv_b58 = base58.b58encode(bytes(fake_keypair.secret())).decode()

# Save to an obvious location (bait)
with open("/home/user/Desktop/My_Private_Keys.txt", "w") as f:
    f.write(f"Private Key: {fake_priv_b58}\n")
    f.write(f"Address: {fake_pubkey}\n")

print("Honeypot wallet created. Fund it with ~0.01 SOL as bait.")
print("Monitor the address. If it ever gets drained, you know someone accessed your system.")

Fund a New Wallet (Test Transaction First)

Always send a small test amount first. Before moving significant funds to a new wallet:

  1. Create the new wallet (save private key securely!)
  2. Send 0.001 SOL as a test transaction
  3. Verify the receiving wallet confirms the transaction (check_balance.py)
  4. If confirmed, send the remaining amount
  5. If NOT confirmed after 5 minutes, stop and investigate -- do not send more
This has saved many people from losing everything. A mistyped address, wrong network, or corrupted key file can be caught by a 0.001 SOL test. Never skip this step.

Non-Negotiable Rules

  • Never store a private key in plain text (unless encrypted drive)
  • Never commit a private key to Git (add *key*.txt to .gitignore)
  • Never paste a private key into a website, Discord bot, or AI chat
  • Never send your full balance in one transaction (always test first)
  • Never share your private key with anyone -- no exceptions
  • Always have at least 3 independent backups of your key
  • Always use the 4-way split method for your main wallet
  • Always set up a honey pot to detect intrusions
Key security done right. Your keys are split, backed up, honeypot-protected, and your funds are spread across multiple wallets. An attacker would need to compromise 4 separate systems AND defeat encryption to steal from you. You are safer than 99% of crypto users.

2 Check Wallet Balance

Once funds have been sent to your wallet, use this script to confirm receipt. It connects to the Solana JSON-RPC API and reads both the SOL balance and any SPL token balances (USDC, USDT, etc.).

2.1 Script: check_balance.py

Reads a wallet address from the command line or uses a default. Returns SOL balance in SOL (converted from lamports) and iterates all token accounts. Download 

#!/usr/bin/env python3
# check_balance.py - Check SOL and SPL token balances

from solana.rpc.api import Client
from solders.pubkey import Pubkey
import sys

# ── Configuration ──────────────────────────────────────────
RPC_URL = "https://api.mainnet-beta.solana.com"
# For testing: "https://api.devnet.solana.com"
# ───────────────────────────────────────────────────────────

KNOW_MINTS = {
    "USDC": "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v",
    "USDT": "Es9vMFrzaCERmJfrF4H2FYD4KCoNkY11McCe8BenwNYB",
}

def get_sol_balance(client, address):
    pubkey = Pubkey.from_string(address)
    resp = client.get_balance(pubkey)
    lamports = resp['result']['value']
    sol = lamports / 1_000_000_000
    return sol, lamports

def get_token_balances(client, address):
    pubkey = Pubkey.from_string(address)
    tokens = []
    try:
        resp = client.get_token_accounts_by_owner_json_parsed(pubkey)
        accounts = resp['result']['value']
        for acct in accounts:
            info = acct['account']['data']['parsed']['info']
            mint = info['mint']
            amount = int(info['tokenAmount']['amount'])
            decimals = info['tokenAmount']['decimals']
            balance = amount / (10 ** decimals)
            symbol = next((k for k, v in KNOW_MINTS.items() if v == mint), mint[:8])
            tokens.append({"symbol": symbol, "mint": mint, "balance": balance})
    except Exception as e:
        print(f"Token fetch error (optional): {e}")
    return tokens

def check_wallet(address):
    client = Client(RPC_URL)
    sol, lamports = get_sol_balance(client, address)
    tokens = get_token_balances(client, address)

    print(f"Wallet: {address}")
    print(f"SOL: {sol:.6f} ({lamports} lamports)")
    if tokens:
        for t in tokens:
            print(f"  {t['symbol']:6s}: {t['balance']:.6f}")
    else:
        print("No SPL tokens found.")
    return {"address": address, "sol": sol, "lamports": lamports, "tokens": tokens}

if __name__ == "__main__":
    addr = sys.argv[1] if len(sys.argv) > 1 else "7g3XzqQBqvZiJAvBiR1Xg5wPZK6KtVfQqL9WzQfZb8cF"
    check_wallet(addr)

3 Transfer SOL

This script loads the wallet keypair from the file created in Step 1, checks the current balance, builds a transfer instruction, signs it, and sends it to the Solana network. The transaction typically confirms in under one second.

Security: This script reads a private key from a plain-text file. In production, never store private keys on disk unencrypted. Use environment variables or a vault service. This example is for learning only.

3.1 Script: transfer.py

Takes three arguments: the wallet file path, the receiver address, and the amount in SOL. Download 

#!/usr/bin/env python3
# transfer.py - Send SOL from your wallet to another address

from solana.rpc.api import Client
from solders.keypair import Keypair
from solders.pubkey import Pubkey
from solders.system_program import transfer, TransferParams
from solders.message import MessageV0
from solders.transaction import VersionedTransaction
from solders.hash import Hash
import base58
import sys

RPC_URL = "https://api.mainnet-beta.solana.com"

def load_keypair_from_file(filepath):
    with open(filepath) as f:
        for line in f:
            if "Private Key (Base58):" in line:
                b58 = line.split(": ", 1)[1].strip()
                break
    keypair = Keypair.from_bytes(base58.b58decode(b58))
    return keypair

def send_sol(sender, receiver_address, amount_sol):
    client = Client(RPC_URL)
    receiver = Pubkey.from_string(receiver_address)
    lamports = int(amount_sol * 1_000_000_000)

    # Get recent blockhash
    bh_resp = client.get_latest_blockhash()
    blockhash = Hash.from_string(bh_resp['result']['value']['blockhash'])

    # Build transfer instruction
    ix = transfer(TransferParams(
        from_pubkey=sender.pubkey(),
        to_pubkey=receiver,
        lamports=lamports,
    ))

    # Compile message and create versioned transaction
    msg = MessageV0.try_compile(
        payer_key=sender.pubkey(),
        instructions=[ix],
        address_lookups=[],
        recent_blockhash=blockhash,
    )
    txn = VersionedTransaction(msg, [sender])

    # Send
    resp = client.send_transaction(txn)
    sig = resp['result']

    print(f"Sent {amount_sol} SOL")
    print(f"To: {receiver_address}")
    print(f"Signature: {sig}")
    print(f"Explorer: https://solscan.io/tx/{sig}")
    return sig

if __name__ == "__main__":
    if len(sys.argv) < 4:
        print("Usage: python3 transfer.py <wallet_file> <to_address> <amount_sol>")
        print('Example: python3 transfer.py "my-new-wallet-key-xxx.txt" "RecvAddr" 0.01')
        sys.exit(1)

    sender = load_keypair_from_file(sys.argv[1])
    send_sol(sender, sys.argv[2], float(sys.argv[3]))

4 Transfer SPL Tokens (USDC, USDT)

Sending SPL tokens (like USDC or USDT) is different from sending SOL. You need to interact with the SPL Token Program and use Associated Token Accounts (ATAs). This script handles that.

4.1 Script: transfer_token.py

Derives the Associated Token Account for sender and receiver, then issues a transfer instruction through the Token Program. Download 

#!/usr/bin/env python3
# transfer_token.py - Send SPL tokens (USDC, USDT, etc.)

from solders.keypair import Keypair
from solders.pubkey import Pubkey
from solders.message import MessageV0
from solders.transaction import VersionedTransaction
from solders.hash import Hash
from solders.associated_token_account import get_associated_token_address
from spl.token.instructions import (
    transfer as spl_transfer,
    TransferParams as SplTransferParams,
)
from spl.token.constants import TOKEN_PROGRAM_ID
from solana.rpc.api import Client
import base58, sys

RPC_URL = "https://api.mainnet-beta.solana.com"

def load_keypair(filepath):
    with open(filepath) as f:
        for line in f:
            if "Private Key (Base58):" in line:
                return Keypair.from_bytes(base58.b58decode(line.split(": ", 1)[1].strip()))

def send_spl_token(sender, receiver_address, token_mint, amount_raw):
    client = Client(RPC_URL)
    receiver = Pubkey.from_string(receiver_address)
    mint = Pubkey.from_string(token_mint)

    # Derive Associated Token Accounts
    sender_ata = get_associated_token_address(sender.pubkey(), mint)
    receiver_ata = get_associated_token_address(receiver, mint)

    # Build transfer instruction
    ix = spl_transfer(SplTransferParams(
        program_id=TOKEN_PROGRAM_ID,
        source=sender_ata,
        dest=receiver_ata,
        owner=sender.pubkey(),
        amount=amount_raw,
    ))

    bh = Hash.from_string(client.get_latest_blockhash()['result']['value']['blockhash'])
    msg = MessageV0.try_compile(sender.pubkey(), [ix], [], bh)
    sig = client.send_transaction(VersionedTransaction(msg, [sender]))['result']

    print(f"Token transfer sent: {sig}")
    return sig

if __name__ == "__main__":
    if len(sys.argv) < 5:
        print("Usage: python3 transfer_token.py <wallet_file> <to_addr> <mint_addr> <raw_amount>")
        print("Example: python3 transfer_token.py wallet.txt RecvAddr EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v 1000000")
        print("Note: raw_amount = amount * 10^decimals. For USDC (6 decimals), 1 USDC = 1000000")
        sys.exit(1)

    sender = load_keypair(sys.argv[1])
    send_spl_token(sender, sys.argv[2], sys.argv[3], int(sys.argv[4]))

Reality Check: Solana Cannot Be Mined Directly

Key fact: Solana is Proof-of-Stake, not Proof-of-Work. You cannot mine SOL directly with GPU or CPU. However, you can mine other coins and convert them to SOL.

Option A: Unmineable (Recommended for Beginners)

Unmineable is a mining pool that lets you mine any mineable coin (Ravencoin, Monero, Ethereum Classic, etc.) and auto-converts your earnings to SOL. You point your miner at their pool, set your SOL wallet address as payout, and they handle the conversion.

  • Supported payout: SOL and 80+ other coins
  • Fee: 1% mining fee
  • Software: Use their "unMiner" app or any standard miner (lolMiner, T-Rex, XMRig, etc.)
  • Minimum payout: Varies by coin price (check their site)
  • Platform: Windows, Linux, macOS

Option B: NiceHash + Manual Conversion

NiceHash auto-switches mining to the most profitable algorithm and pays you in Bitcoin. You then manually swap BTC to SOL on any exchange (Binance, Backpack, etc.).

  • Payout: Bitcoin (then swap to SOL)
  • Software: NiceHash QuickMiner or NiceHash Miner
  • Fee: 2% mining fee
  • Platform: Windows, Linux (NHOS)

Option C: DePIN Networks (GPU Rental for SOL)

Newer projects pay you directly in SOL for contributing idle GPU power:

  • HIVE Network (hivenetwork.fun) -- contribute idle GPU power, earn SOL rewards automatically. Payouts sent directly to your Solana wallet with no manual conversion.
  • io.net -- decentralized GPU network on Solana. Rent out GPU power for AI/ML workloads, get paid in IO (Solana token).
  • IDLE Protocol -- turn any idle resource (GPU, API, agent) into a paid endpoint on Solana.

9 How to Become a Good Trading Agent

Having a bot that can place trades is not enough. A good trading agent is one that consistently makes profitable decisions. This requires education, testing, and discipline. The difference between a losing agent and a winning one is the process -- not the code.

Step 1: Study Existing Trading Agents

Before writing your own strategies, study what other trading agents are doing. The AI Repos Agent Catalog is a curated directory of AI agents including trading bots, market analyzers, and automated strategies. Browse the catalog to understand different approaches, architectures, and what works in production.

ai-repos.com/#/agent-catalog -- bookmark this. Review agents regularly to spot new strategies and techniques.

Step 2: Learn the Algorithms

You do not need to be a quant, but you must understand the basic building blocks:

  • Moving Averages (SMA, EMA) -- trend direction and smoothing
  • RSI (Relative Strength Index) -- overbought/oversold conditions
  • MACD -- trend momentum and signal crossovers
  • Bollinger Bands -- volatility-based support/resistance
  • VWAP (Volume-Weighted Average Price) -- fair value with volume
  • Order Book Imbalance -- bid/ask pressure for short-term moves
  • Funding Rate Analysis -- sentiment in perpetuals markets

Step 3: Learn the Strategies

  • Trend Following -- ride the trend until it reverses
  • Mean Reversion -- buy dips, sell pumps within a range
  • Breakout Trading -- enter when price breaks key levels with volume
  • Grid Trading -- place multiple buy/sell orders in a range
  • Scalping -- very short holds (seconds to minutes), many small wins
  • Momentum Trading -- enter on strong directional moves

Step 4: Backtest Everything

Never deploy a strategy without backtesting it first. Use historical data to simulate how the strategy would have performed. Backtest over different market conditions (bull, bear, sideways).

  • Use Hyperliquid historical data or free APIs (Binance, Bybit) for price data
  • Write a backtester in Python that simulates trades with 0.1s tick precision
  • Track: win rate, average win/loss, max drawdown, Sharpe ratio
  • Target: win rate above 54% -- this is your baseline for a profitable edge
  • Look for constant gains, not a few big wins with many small losses
54% win rate target: With a 1:1 risk-reward ratio, 54% wins means you are profitable. If your average win is larger than your average loss, even 40% wins can be profitable. But for scalping strategies, aim for >54% win rate with quick small wins.

Step 5: Only Quick In-And-Outs (Max 1 Day Hold)

As an AI agent trading perpetuals, never hold longer than 1 day. Overnight funding fees eat your profits. Holding longer also exposes you to unnecessary risks:

  • Gap risk -- price jumps while you are not watching
  • Funding cost -- perpetuals have 8-hour funding payments that add up
  • News risk -- unexpected events can blow through stops
  • Opportunity cost -- capital locked in a position that goes nowhere

Quick in-and-outs (minutes to hours) let you compound frequently and reduce uncertainty. This is how scalping bots achieve steady PnL.

Step 6: Always Use Stop Loss + Take Profit

Every trade must have both. This is non-negotiable for a profitable agent.

  • Stop Loss: Define the maximum you are willing to lose on a trade. A good starting point is 0.5-1% of your account per trade.
  • Take Profit: Define your exit target before entering. Use 1.5x to 3x your stop loss distance for a positive risk-reward ratio.
  • Trailing Stop: Let profits run by moving the stop loss up as the price moves in your favor.

Step 7: Higher Conviction = Higher Probability

Only take trades where multiple indicators agree. If your strategy needs 3 signals to enter and only 2 are present -- skip the trade. The more confirmation you have, the higher the probability of success.

A good rule: if you cannot clearly explain why you are taking a trade, do not take it. Let the market come to you.

Step 8: Track Everything, Iterate Constantly

Log every trade. Analyse your performance weekly. Cut strategies that lose. Double down on strategies that win. The market changes -- your strategies must adapt.

How-To: Read Bitcoin Price Chart (Candlestick Analysis)

A candlestick tells you everything about price action in a time period. Each candle has 4 key values: Open, High, Low, Close (OHLC). The body shows open-to-close movement. The wicks (shadows) show the full range. Download 

#!/usr/bin/env python3
# btc_candles.py - Analyse candlestick patterns from live data

import urllib.request, json
from datetime import datetime

# Fetch 50 hourly candles
url = "https://api.binance.com/api/v3/klines?symbol=BTCUSDT&interval=1h&limit=50"
with urllib.request.urlopen(url, timeout=10) as r:
    candles = json.loads(r.read())

print("=== How to Read a Candlestick ===")
print("Each candle: [Open, High, Low, Close, Volume]")
print("  Green candle: Close > Open (bullish, price went up)")
print("  Red candle:   Close < Open (bearish, price went down)")
print("  Wick (shadow): High-Low range (volatility)")
print("  Body: |Open-Close| (momentum strength)")
print()

# Analyse each candle
bullish_count = 0
bearish_count = 0
total_range = 0

print(f"{'Time':20s} {'Dir':4s} {'Open':>10s} {'Close':>10s} {'High':>10s} {'Low':>10s} {'Body':>8s} {'Wick':>8s} {'Vol':>10s}")
print("-" * 90)

for c in candles:
    t = datetime.fromtimestamp(c[0]/1000).strftime("%m-%d %H:%M")
    o, h, l, cl, v = float(c[1]), float(c[2]), float(c[3]), float(c[4]), float(c[5])

    if cl > o:
        direction = "BUY"
        bullish_count += 1
    elif cl < o:
        direction = "SELL"
        bearish_count += 1
    else:
        direction = "---"

    body = abs(cl - o)
    wick = h - l
    total_range += wick

    if body > 0:  # skip empty candles
        body_pct = (body / wick) * 100 if wick > 0 else 0
        print(f"{t:20s} {direction:4s} {o:>10.2f} {cl:>10.2f} {h:>10.2f} {l:>10.2f} {body:>8.2f} {wick:>8.2f} {v:>10.0f}")

print("-" * 90)
print(f"\n=== Pattern Summary (last {len(candles)} hours) ===")
print(f"Bullish candles: {bullish_count} ({bullish_count/len(candles)*100:.1f}%)")
print(f"Bearish candles: {bearish_count} ({bearish_count/len(candles)*100:.1f}%)")
print(f"Avg range/volatility: \${total_range/len(candles):.2f}")

# Detect doji (body is very small = indecision)
doji_count = sum(1 for c in candles if abs(float(c[4])-float(c[1])) < float(c[2])-float(c[3])*0.05)
print(f"Doji candles (indecision): {doji_count}")

# Detect engulfing (potential reversal signal)
engulfing = 0
for i in range(1, len(candles)):
    prev_o, prev_c = float(candles[i-1][1]), float(candles[i-1][4])
    curr_o, curr_c = float(candles[i][1]), float(candles[i][4])
    # Bullish engulfing: prev red, curr green, curr body covers prev body
    if prev_c < prev_o and curr_c > curr_o and curr_o < prev_c and curr_c > prev_o:
        engulfing += 1
    # Bearish engulfing: prev green, curr red, curr body covers prev body
    if prev_c > prev_o and curr_c < curr_o and curr_o > prev_c and curr_c < prev_o:
        engulfing += 1
print(f"Engulfing patterns (reversal): {engulfing}")
